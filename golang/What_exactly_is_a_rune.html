Understanding Runes in Go

In Go, a rune is a data type that represents a Unicode code point.

What exactly is a rune?

  • A rune is an alias for int32.
  • It represents a single Unicode character.
  • Go strings are UTF-8 encoded, so a character can be multiple bytes.

Why use runes?

The string type in Go is a sequence of bytes, not characters. Accessing string elements by index accesses bytes, not full characters. To work with actual Unicode characters, use runes.

Example Code

package main

import "fmt"

func main() {
    s := "Hello, 世界"

    // Access bytes (not characters)
    fmt.Println("Bytes:")
    for i := 0; i < len(s); i++ {
        fmt.Printf("%x ", s[i])
    }
    fmt.Println()

    // Access runes (characters)
    fmt.Println("Runes:")
    for i, r := range s {
        fmt.Printf("%d: %c\n", i, r)
    }
}

Output

Bytes: 48 65 6c 6c 6f 2c 20 e4 b8 96 e7 95 8c Runes: 0: H 1: e 2: l 3: l 4: o 5: , 6: 7: 世 10: 界

Summary

rune is an int32 representing a Unicode code point. Use runes to properly handle Unicode characters in Go strings.
