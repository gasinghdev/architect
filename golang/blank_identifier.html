Blank Identifier in Go

The blank identifier in Go is represented by an underscore (_). It acts as a write-only variable—you can assign values to it, but you cannot read from it. It is a useful feature when you want to ignore certain values without causing compiler errors.

1. Ignoring Unused Return Values

When a function returns multiple values, you can use _ to ignore the ones you don't need.

num, _ := strconv.Atoi("123")
fmt.Println(num) // Only the integer is used, error is ignored

2. Ignoring Values in Loops

When iterating over a collection with range, you may not always need both the index and the value.

words := []string{"go", "is", "fun"}
for _, word := range words {
    fmt.Println(word) // index is ignored
}

3. Importing Packages for Side Effects

The blank identifier can be used in import statements when you want to import a package only for its initialization side effects.

import _ "net/http/pprof"

Here, the pprof package is imported only to enable profiling, even though we don’t reference it directly in the code.

Summary

  • _ is the blank identifier in Go.
  • It discards values you don’t want to use.
  • Commonly used to ignore return values, loop variables, or import side-effect packages.
