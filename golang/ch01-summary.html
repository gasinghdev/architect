Go Programming - Chapter 01

Packages in Go

A package is a collection of related functions, types, and other code grouped together for better organization and reusability. To use a package, you must explicitly import it into your file.

import "fmt"

Strings

A string in Go is a sequence of bytes that usually represents text characters. Strings are immutable, meaning they cannot be changed after creation.

message := "Hello, Go!"

Runes

A rune represents a single Unicode code point, essentially one text character.

var ch rune = 'A'

Numeric Types

Go’s two most common numeric types are:

  • int: Stores whole numbers.
  • float64: Stores decimal numbers.

Boolean Type

The bool type holds values true or false, commonly used in conditions.

var isActive bool = true

Variables

A variable is a named piece of storage that can hold a value of a specified type.

var age int = 25

Zero Values

If a variable is declared without a value, Go assigns it a zero value based on its type.

  • int → 0
  • float64 → 0.0
  • string → ""
  • bool → false

Short Variable Declaration

Go allows you to declare and initialize a variable in a single step using :=.

name := "Alice"

Exported Names

Identifiers starting with a capital letter are exported and accessible outside their package, while lowercase names are private.

fmt.Println("Hello")

go fmt

The go fmt command reformats source code to follow Go’s standard formatting rules.

go build

The go build command compiles Go source code into a binary executable.

go build main.go

go run

The go run command compiles and runs Go programs without leaving an executable file behind.

go run main.go

Multiple Return Values

In Go, functions and methods can return multiple values, unlike many other languages. This feature is most often used to return both a result and an error value.

Example: Reading Input

input, err := reader.ReadString('\n')
if err != nil {
    fmt.Println("Error:", err)
}

Example: String to Integer Conversion

num, err := strconv.Atoi("123")
if err != nil {
    fmt.Println("Conversion failed:", err)
} else {
    fmt.Println("Converted number:", num)
}

Key Points:

  • First return value → main result.
  • Second (or later) return value → often an error.
  • This makes error handling explicit and reliable.
