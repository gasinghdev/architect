1) Core & Advanced Java

JVM • GC • Concurrency
Be ready to explain not just what you use, but why: design choices, trade‑offs, and performance impact.
Java Fundamentals: OOP, Collections, Generics, Streams, Lambdas, Functional Interfaces.
Memory & GC: heap vs stack, escape analysis, GC types (G1/ZGC), tuning flags, profiling.
Concurrency: threads, locks, CAS, Executors, CompletableFuture, Fork/Join, deadlock/ livelock.
JVM Internals: class loading, JIT/HotSpot, bytecode, startup vs throughput trade‑offs.
Modern Java: Streams best practices, Optionals, Records, Sealed classes, Virtual Threads (Loom).

2) Frameworks & Middleware

Spring • JPA • Messaging
Spring Core: IoC, AOP, events, bean scopes, configuration properties, validation.
Spring Boot: auto‑configuration, profiles, Actuator, configuration layering, graceful shutdown.
Spring Data JPA/Hibernate: entity mapping, caching, N+1, lazy vs eager, batch writes, 2nd‑level cache.
Spring Cloud: config server, discovery, API gateway, circuit breaker, Sleuth/Zipkin.
Messaging: Kafka/RabbitMQ/JMS fundamentals; exactly‑once semantics myth vs reality, idempotency.

3) Microservices Architecture

Patterns • Resilience • Consistency
Design Patterns: Saga, CQRS, Event Sourcing, Strangler Fig, BFF, Sidecar.
API Design: REST maturity, pagination, versioning, GraphQL basics, gRPC for internal comms.
Resilience: retry, circuit breaker, bulkhead, timeout budgets, rate limiting, backpressure.
Service Mesh: Istio/Linkerd basics, mutual TLS, traffic shaping, observability.
Distributed Transactions: 2PC pitfalls, Outbox pattern, inbox/ dedupe, eventual consistency.

4) Cloud & Deployment

Kubernetes • CI/CD • Serverless
Cloud Platforms: AWS/Azure/GCP foundations—identity, networking, storage, compute, costing.
Containers & Orchestration: Docker images, security, K8s workloads, services/ingress, Helm.
CI/CD: pipelines, artifact management, blue/green & canary, GitOps (ArgoCD/Flux).
Serverless: Lambda/Functions fundamentals, cold starts, when to use vs containers.

5) Databases & Storage

SQL • NoSQL • Caching
Relational: indexing, query plans, transactions, isolation, deadlocks, partitioning.
NoSQL: document (MongoDB), wide‑column (Cassandra), key‑value (Redis), consistency models.
Caching: local vs distributed, TTL/refresh‑ahead, cache stampede, write‑through/behind.
Data Modeling: normalization vs denormalization, sharding strategies, data retention.

6) System Design & Architecture

Scale • Patterns • Security
Scalability: horizontal vs vertical, load balancing, autoscaling, CAP & PACELC, backpressure.
Design Patterns: Factory, Singleton, Builder, Observer, Proxy, Adapter, Strategy.
Enterprise Integration: EDA, message brokers, choreography vs orchestration, idempotency.
Edge & Gateways: API Gateway vs Reverse Proxy, rate limiting, authn/z, WAF basics.
Security: OAuth2/OIDC, JWT, SAML, TLS/SSL, CSRF/XSS/SQLi mitigations, secrets mgmt.

7) DevOps & Observability

Metrics • Traces • SLOs
Monitoring & Logging: Prometheus, Grafana, ELK/Splunk; log correlation, structured logging.
Tracing: OpenTelemetry, Jaeger/Zipkin; spans, sampling, trace‑based alerting.
Health & Metrics: Actuator/Micrometer, RED/USE metrics, SLI/SLO/SLA, error budgets.
Testing: unit/integration/contract/E2E; performance with JMeter/Gatling; chaos testing.

8) Best Practices

Quality • Maintainability
Clean Code & SOLID: layering, ports & adapters, domain‑driven design lite, readability.
12‑Factor Apps: config, logs, disposability, statelessness, admin processes.
Backward Compatibility: API versioning, schema migration, dark launches, feature flags.
Documentation: ADRs, architecture diagrams (C4), runbooks, on‑call handoffs.

Practice Prompts

Storytelling • Trade‑offs
High Availability Payment System (15–20 min)
  1. Requirements: 99.99% availability, PCI concerns, burst traffic (5× on sale days), multi‑region read/write.
  2. Sketch: API Gateway → Auth → Orchestrator → Payment Providers (idempotent), Outbox, Sagas for refund.
  3. Deep dive: data consistency, failure modes, retries/backoff, monitoring, canary, rollback plan.
Data Consistency in Microservices

Compare 2PC vs Saga. Explain Outbox pattern, dedupe keys, exactly‑once effect vs delivery, and recovery strategies.

Scale to Millions of RPS

Discuss caching layers, async processing, connection pooling, reactive vs blocking, backpressure, rate limiting, autoscaling, cost.

Security Deep Dive

Contrast OAuth2/OIDC vs SAML; JWT pitfalls (size/rotation/blacklist); zero‑trust, secrets rotation, SBOM & supply‑chain risks.

Observability Story

Tell a war story: how metrics + traces revealed a performance regression, and how you fixed it with RED/USE dashboards.

Pro Tips

Interview Ready
  • Frame answers as: Context → Options → Trade‑offs → Decision → Impact.
  • Use numbers (latency, throughput, cost) and failure modes to show depth.
  • Bring a consistent reference design (C4 level 1–3) you can adapt for different prompts.
